オブジェクトの作り方

  • オブジェクトの設計図を作る
  • 設計図 → クラス
  • 1つの設計図（クラス）から多数のオブジェクトを生成
  • オブジェクトの1つ1つは異なる値をフィールドに保存
同じ種類のスマートフォン
色，メモリ容量，機種番号等異なる．
しかし，元の設計図は同じ
クラス（設計図）→オブジェクト（もの）
オブジェクト=クラスのインスタンス

クラスの定義

  1. ブロックを作成
  2. フィールドの追加
  3. コンストラクタの作成とフィールドへの代入
  4. メソッドの追加

ブロックを作成

	    
	      class JitterBug {

	      }
  • JitterBugという名前のクラスを作成するという宣言
  • クラス名は大文字で始めることを推奨

フィールドの追加

	    
	      class JitterBug {
	        float x;
	        float y;
	        int diameter;
	        float speed = 2.5;
	      }
  • コンストラクタで値を代入するフィールドを決定

コンストラクタの作成とフィールドの代入

	    
	      class JitterBug {
	        float x;
	        float y;
	        int diameter;
	        float speed = 2.5;
	        JitterBug(float tempX, float tempY, int tempDiameter) {
	          x = tempX;
	          y = tempY;
	          diameter = tempDiameter;
	        }
	      }
  • コンストラクタはクラスの初期化を担当
  • コンストラクタ名はクラスと同じ名前

メソッドmove()の追加

	    
class JitterBug {
  float x;
  float y;
  int diameter;
  float speed = 2.5;
  JitterBug(float tempX, float tempY, int tempDiameter) {
    x = tempX;
    y = tempY;
    diameter = tempDiameter;
  }
  void move() {
    x += random(-speed, speed);
    y += random(-speed, speed);
  }
}
  • move()というメソッドを追加
  • move()はオブジェクトの移動を定義

メソッドdisplay()の追加

	    
class JitterBug {
  float x;
  float y;
  int diameter;
  float speed = 2.5;
  JitterBug(float tempX, float tempY, int tempDiameter) {
    x = tempX;
    y = tempY;
    diameter = tempDiameter;
  }
  void move() {
    x += random(-speed, speed);
    y += random(-speed, speed);
  }
  void display() {
    ellipse(x, y, diameter, diameter);
  }
}
  • display()というメソッドを追加
  • display()はオブジェクトの表示を定義

複数のオブジェクトを作る

09_02

	    
JitterBug jit;
JitterBug bug;
void setup() {
  size(480, 120);
  jit = new JitterBug(width * 0.33, height/2, 50);
  bug = new JitterBug(width * 0.66, height/2, 10);

void draw() {
  jit.move();
  jit.display();
  bug.move();
  bug.display();
}

タブ機能

  • PDEにはタブ機能が存在
  • 基本はフォルダとファイルのペア
  • コードのファイル名拡張子は .pde
  • 設定変更で表示可能
  • フォルダ内に複数のファイル→タブに表示
  • 共通利用するクラスを別ファイルで保存
  • 複数コードからなる開発が可能
  • クラスの開発を分担可能
